## Why Drizzle?

### SQL-like Approach
**If you know SQL, you know Drizzle.**

Other ORMs tend to deviate from or abstract away SQL, leading to a double learning curve. Drizzle is the opposite - it embraces SQL while providing TypeScript safety.

### Headless Architecture
Drizzle is designed to be a thin typed layer on top of SQL. You have full control over your queries while enjoying the benefits of type safety and developer experience.

## Supported Databases

- **PostgreSQL** - Full support with pg, postgres.js, node-postgres
- **MySQL** - Compatible with MySQL 5.7+ and MySQL 8+
- **SQLite** - Works with better-sqlite3, bun:sqlite, and more
- **SingleStore** - Full compatibility with SingleStore databases

## Getting Started

Install Drizzle ORM and Drizzle Kit:

```bash
npm i drizzle-orm
npm i -D drizzle-kit
```

Define your schema:

```typescript
import { pgTable, serial, text, integer } from 'drizzle-orm/pg-core';

export const users = pgTable('users', {
  id: serial('id').primaryKey(),
  name: text('name').notNull(),
  age: integer('age'),
});
```

Query your database:

```typescript
import { drizzle } from 'drizzle-orm/node-postgres';
import { users } from './schema';

const db = drizzle(process.env.DATABASE_URL);

// SQL-like query
const allUsers = await db.select().from(users);

// Relational query
const result = await db.query.users.findMany();
```
